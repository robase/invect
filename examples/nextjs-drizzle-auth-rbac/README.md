# Acme Dashboard — Next.js + Drizzle + Invect

A realistic admin dashboard for a pretend startup ("Acme"), demonstrating how
to add **Invect** workflow automation into an **existing** Next.js application
that already has its own Drizzle schema.

## What this example shows

1. **Pre-existing app schema** — customers, products, orders, team members
2. **Invect added as one page** — the "Workflows" tab inside the dashboard
3. **CLI-driven installation** — `npx invect generate` merges Invect tables
   into the existing schema, then `drizzle-kit push` applies them
4. **Auth + RBAC plugins** — better-auth for login, RBAC for flow access control

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 15 (App Router) |
| Database | PostgreSQL 16 via Docker |
| ORM | Drizzle ORM |
| Auth | better-auth + @invect/user-auth plugin |
| RBAC | @invect/rbac plugin |
| Workflow | @invect/core + @invect/nextjs + @invect/frontend |

## Quick Start

```bash
# From the monorepo root
pnpm install

cd examples/nextjs-drizzle-auth-rbac

# 1. Set up environment
cp .env.example .env
# Edit .env — set BETTER_AUTH_SECRET at minimum

# 2. Start PostgreSQL
docker compose up -d

# 3. Create app database tables (customers, products, orders, team)
pnpm db:push

# 4. Seed the database with sample data
pnpm db:seed

# 4. Generate Invect schema (merges core + plugin tables)
pnpm invect:generate

# 5. Uncomment the Invect export in db/schema.ts
#    (the line: export * from './schema-postgresql')

# 6. Push all tables (app + Invect) to the database
pnpm db:push

# 7. Start the dev server
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) → **Open Dashboard** → **Workflows**.

## How the Installation Flow Works

This is the typical flow for adding Invect to an existing app:

1. **You already have** a Drizzle schema (`db/app-schema.ts`) and a running app
2. **Install** `@invect/core`, `@invect/nextjs`, `@invect/frontend`
3. **Create** `lib/invect.config.ts` pointing to your Invect config
4. **Run** `npx invect generate --output db --dialect postgresql --yes`
   - The CLI reads your config, discovers plugins (auth, RBAC)
   - Merges Invect's core tables + plugin tables
   - Writes `db/schema-postgresql.ts`
5. **Re-export** the generated schema from `db/schema.ts`
6. **Run** `drizzle-kit push` to create the tables in your database
7. **Mount** the Invect API handler and UI component

## E2E Test

The root Playwright suite contains a dedicated example test at
`playwright/tests/examples/nextjs-drizzle-auth-rbac-installation.spec.ts`
that validates this entire installation flow end-to-end:

```bash
pnpm test:e2e
```

The test:
1. Starts from a clean database (app tables only)
2. Runs `npx invect generate` and verifies schema files are created
3. Patches `db/schema.ts` to include the generated Invect tables
4. Runs `drizzle-kit push` to apply the schema
5. Starts the Next.js dev server
6. Navigates the dashboard and verifies pages render
7. Opens the Workflows page and verifies Invect loads
