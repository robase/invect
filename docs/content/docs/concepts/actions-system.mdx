The **actions system** is how all node types (except Agent) and all agent tools are defined. A single `defineAction()` call creates both a flow node and an agent tool.

## defineAction()

Every action is a self-contained file that defines:

```ts title="pkg/core/src/actions/core/jq.ts"
import { defineAction } from '../define-action';
import { CORE_PROVIDER } from '../providers';
import { z } from 'zod/v4';

export const jqAction = defineAction({
  id: 'core.jq',
  name: 'JQ Transformation',
  description: 'Query and transform JSON data using JQ syntax',
  provider: CORE_PROVIDER,

  params: {
    schema: z.object({
      query: z.string(),
    }),
    fields: [{ name: 'query', label: 'JQ Query', type: 'code', required: true }],
  },

  async execute(params, context) {
    const result = jq.run(params.query, context.incomingData);
    return { success: true, output: result };
  },
});
```

## Action definition fields

| Field           | Required | Description                                                       |
| --------------- | -------- | ----------------------------------------------------------------- |
| `id`            | Yes      | Unique string identifier (e.g., `core.jq`, `gmail.send_message`)  |
| `name`          | Yes      | Human-readable name for the UI                                    |
| `description`   | Yes      | What the action does (shown in UI and to LLMs for tool selection) |
| `provider`      | Yes      | Provider grouping (determines palette category)                   |
| `params.schema` | Yes      | Zod schema for input validation                                   |
| `params.fields` | Yes      | UI field definitions (label, type, required)                      |
| `execute`       | Yes      | Async function that runs the action                               |
| `credential`    | No       | Credential requirements (OAuth2 provider, required flag)          |

## Execution context

The `execute` function receives validated params and a context object:

```ts
async execute(params, context) {
  // params — Zod-validated and template-resolved config
  // context.credential — OAuth2 tokens or API key (auto-refreshed)
  // context.incomingData — upstream node outputs
  // context.flowInputs — original flow inputs
  // context.logger — structured logger
  // context.functions — service functions:
  //   .submitPrompt() — call an LLM
  //   .getCredential(id) — fetch a credential
  //   .markDownstreamNodesAsSkipped() — for flow control nodes
  // context.flowRunState — current flow run state
}
```

## Action results

Actions return an `ActionResult`:

```ts
// Simple result
return { success: true, output: { data: 'hello' } };

// Error result
return { success: false, error: 'Something went wrong' };

// Multi-output (for flow control nodes like if/else)
return {
  success: true,
  output: data,
  outputVariables: {
    true_output: data,
    false_output: null,
  },
};
```

## Dual registration

During `Invect.initialize()`, every action is registered in two places:

1. **ActionRegistry** — used by the flow execution engine to run nodes
2. **AgentToolRegistry** — used by the Agent node to provide tools to LLMs

The conversion from action to agent tool happens automatically via `toAgentToolDefinition()`:

- `id` → tool ID
- `name` → tool name
- `description` → tool description (sent to the LLM)
- `params.schema` → JSON Schema input definition

## Providers

Actions are grouped by **provider** for UI organization:

```ts
export const GMAIL_PROVIDER: ProviderDef = {
  id: 'gmail',
  name: 'Gmail',
  icon: Mail, // Lucide icon
};
```

## Adding a custom action

### 1. Create the action file

```ts title="my-actions/weather.ts"
import { defineAction } from '@invect/core';
import { z } from 'zod/v4';

export const weatherAction = defineAction({
  id: 'custom.get_weather',
  name: 'Get Weather',
  description: 'Fetch current weather for a city',
  provider: { id: 'custom', name: 'Custom', icon: Cloud },

  params: {
    schema: z.object({
      city: z.string(),
    }),
    fields: [{ name: 'city', label: 'City', type: 'text', required: true }],
  },

  async execute(params, context) {
    const res = await fetch(
      `https://api.weather.com/current?city=${encodeURIComponent(params.city)}`,
    );
    const data = await res.json();
    return { success: true, output: data };
  },
});
```

### 2. Register it

Add the action to the `allBuiltinActions` array in `pkg/core/src/actions/index.ts`, or register it via a plugin's `actions` array.

### 3. Use it

The action appears in the node palette and is available as an agent tool — no additional wiring needed.

<Callout type="info">
  For actions that only make sense as agent tools (not flow nodes), create a [standalone
  tool](/docs/agents/tools#option-b-standalone-tool-utility-only) instead.
</Callout>
