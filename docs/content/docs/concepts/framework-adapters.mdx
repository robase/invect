---
title: Framework Adapters
description: How Express, NestJS, and Next.js adapters wrap the core.
---

Each framework adapter follows the same principle: create an `Invect` instance, initialise it, and map HTTP routes to its methods. The adapters are intentionally thin — all logic lives in `@invect/core`.

## Express

**Pattern**: Async factory function returning an Express `Router`.

```ts
import { createInvectRouter } from '@invect/express';

const router = await createInvectRouter(config);
app.use('/invect', router);
```

Inside `createInvectRouter`:

1. Creates an `Invect` instance
2. Calls `await core.initialize()` (database, plugins, actions, tools)
3. Starts batch polling and cron scheduler
4. Creates an Express router with ~30 routes
5. Each route extracts request data, calls `core.<method>()`, returns the result

The returned router is a standard Express router — mount it at any path, apply middleware, use with other Express routes.

## NestJS

**Pattern**: NestJS dynamic module with dependency injection.

```ts
import { InvectModule } from '@invect/nestjs';

@Module({
  imports: [InvectModule.forRoot(config)],
})
export class AppModule {}
```

Inside `InvectModule.forRoot`:

1. Creates a provider that instantiates and holds a singleton `Invect` core
2. Registers `InvectService` (injectable wrapper around the core)
3. Registers `InvectController` (HTTP endpoints)
4. Exports the provider and service for injection in other modules

The controller delegates every endpoint to `InvectService.getCore().<method>()`. For async config loading, use `InvectModule.forRootAsync()` with `useFactory`.

To access the core from your own services, inject `InvectService`:

```ts
@Injectable()
export class MyService {
  constructor(private invect: InvectService) {}

  getCore() { return this.invect.getCore(); }
}
```

## Next.js

**Pattern**: Handler factory for catch-all App Router routes.

```ts
import { createInvectHandler } from '@invect/nextjs';

const handler = createInvectHandler(config);

export const GET = handler.GET;
export const POST = handler.POST;
```

Inside `createInvectHandler`:

1. Creates a singleton `Invect` instance (lazy-initialized on first request)
2. Returns an object with GET, POST, PATCH, PUT, DELETE handlers
3. Each handler parses the catch-all route segments to determine the Invect endpoint
4. Maps the route to the corresponding `core.<method>()` call

The catch-all route pattern (`[...invect]`) means all sub-paths are handled by the handler. Mount it at `app/api/invect/[...invect]/route.ts`.

## Choosing an adapter

The adapters are functionally identical — they expose the same API endpoints backed by the same core logic. Choose based on your existing framework:

| Framework | Use when |
|-----------|----------|
| Express | Standalone backend, or adding Invect to an existing Express app |
| NestJS | Existing NestJS application, need DI integration |
| Next.js | Adding Invect to a Next.js app with App Router |

All three support the same features: flows, credentials, OAuth2, plugins, cron, webhooks, agents.
