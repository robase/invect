The auth plugin wraps [Better Auth](https://better-auth.com) to provide user login, session management, and admin-only user creation. It's a backend + frontend package.

## Installation

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">```bash pnpm add @invect/user-auth ```</Tab>
  <Tab value="npm">```bash npm install @invect/user-auth ```</Tab>
  <Tab value="yarn">```bash yarn add @invect/user-auth ```</Tab>
  <Tab value="bun">```bash bun add @invect/user-auth ```</Tab>
</Tabs>

## Backend setup

```ts title="invect.config.ts"
import { defineConfig } from '@invect/core';
import { authentication } from '@invect/user-auth';

export const invectConfig = defineConfig({
  database: { type: 'sqlite', connectionString: 'file:./dev.db' },
  plugins: [
    authentication({
      betterAuthOptions: {
        secret: process.env.BETTER_AUTH_SECRET,
      },
      globalAdmins: [{ email: 'admin@example.com', pw: 'changeme', name: 'Admin' }],
    }),
  ],
});
```

The `globalAdmins` array seeds admin users on startup. These are the only users that can log in initially ŌĆö sign-up is disabled in the UI. Subsequent users are created by admins through the user management interface.

After adding the plugin, regenerate and apply the schema:

```bash
npx invect-cli generate
npx invect-cli migrate --push
```

## What it does

**Backend:**

- `onRequest` hook resolves the session from cookies on every API request
- `onAuthorize` hook enforces that the user is logged in
- Endpoints for sign-in, sign-out, session management, and user CRUD (proxied to Better Auth)
- Admin user seeding on startup from `globalAdmins`

**Frontend:**

- `AuthProvider` ŌĆö React context providing session state
- `AuthGate` ŌĆö renders children only when authenticated, shows sign-in page otherwise
- `SignInPage` ŌĆö pre-built sign-in form
- `UserButton` ŌĆö avatar with sign-out menu
- `UserManagement` ŌĆö admin panel for creating/managing users

## Frontend integration

The recommended approach uses `AuthenticatedInvect`, which composes the auth layer around the editor:

```tsx
import { Invect, InvectShell } from '@invect/ui';
import { AuthenticatedInvect } from '@invect/user-auth/ui';
import '@invect/ui/styles';

<AuthenticatedInvect
  apiBaseUrl="/api/invect"
  basePath="/invect"
  InvectComponent={Invect}
  ShellComponent={InvectShell}
  theme="light"
/>;
```

When `ShellComponent` is provided, the sign-in page renders inside the Invect CSS scope ŌĆö so it uses the same theme tokens and looks consistent with the editor.

## Environment variables

| Variable                | Required | Description                          |
| ----------------------- | -------- | ------------------------------------ |
| `BETTER_AUTH_SECRET`    | Yes      | Session encryption secret            |
| `INVECT_ADMIN_EMAIL`    | No       | Alternative to `globalAdmins` config |
| `INVECT_ADMIN_PASSWORD` | No       | Alternative to `globalAdmins` config |
