Invect encrypts all stored credentials (API keys, OAuth2 tokens, database passwords) using **AES-256-GCM** before writing them to the database.

## Encryption key

The `INVECT_ENCRYPTION_KEY` environment variable must be a **base64-encoded 32-byte key**. Generate one:

```bash
npx invect-cli secret
```

This uses `crypto.randomBytes(32)` to generate a cryptographically secure key and outputs it as base64.

Alternatively:

```bash
openssl rand -base64 32
```

<Callout type="error">
  **Never commit the encryption key to source control.** Store it securely in your environment
  variables, secrets manager, or deployment platform.
</Callout>

## How it works

When a credential is created:

1. A random 12-byte **IV** (initialization vector) is generated
2. A random 16-byte **auth tag** is generated via GCM
3. The credential data is encrypted with AES-256-GCM using the key + IV
4. The IV, auth tag, and ciphertext are stored together in the database

When a credential is read:

1. The IV and auth tag are extracted
2. The ciphertext is decrypted with AES-256-GCM
3. The plaintext credential data is returned

## What's encrypted

| Data                        | Encrypted | Notes                                  |
| --------------------------- | --------- | -------------------------------------- |
| API keys                    | Yes       | OpenAI, Anthropic, HTTP API keys       |
| OAuth2 access tokens        | Yes       | Refreshed tokens are re-encrypted      |
| OAuth2 refresh tokens       | Yes       |                                        |
| OAuth2 client secrets       | Yes       | Required for token refresh             |
| Database connection strings | Yes       | PostgreSQL, MySQL credentials          |
| Credential names            | No        | Stored in plaintext for search/display |
| Credential types            | No        | Stored in plaintext                    |

## Key rotation

To rotate the encryption key:

1. Decrypt all credentials with the old key
2. Re-encrypt with the new key
3. Update `INVECT_ENCRYPTION_KEY`

<Callout type="info">
  There is no built-in key rotation command yet. You would need to implement this in a migration
  script that reads all credentials, decrypts with the old key, and re-encrypts with the new key.
</Callout>

## Missing key behavior

If `INVECT_ENCRYPTION_KEY` is not set:

- Creating credentials will fail with an error
- Existing encrypted credentials cannot be decrypted
- The server will start but credential operations will be unavailable

Always ensure the key is set before starting Invect in production.
