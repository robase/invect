title: Use HTTP Requests

---

The HTTP Request node (`http.request`) makes HTTP calls to external APIs and returns the response.

## Add an HTTP Request node

1. Drag **HTTP Request** from the HTTP provider onto the canvas.
2. Double-click to configure.

## Configuration fields

| Field | Required | Description |
|-------|----------|-------------|
| **URL** | Yes | Request URL. Supports `{{ }}` templates. |
| **Method** | Yes | `GET`, `POST`, `PUT`, `PATCH`, `DELETE`. |
| **Headers** | No | Key-value headers. Values support templates. |
| **Body** | No | Request body (for POST/PUT/PATCH). Supports templates. |
| **Credential** | No | Optional credential for authenticated requests. |

## Examples

### Simple GET request

```
URL: https://api.example.com/users/{{ input.userId }}
Method: GET
```

### POST with JSON body

```
URL: https://api.example.com/messages
Method: POST
Headers: { "Content-Type": "application/json" }
Body: { "text": "{{ input.message }}", "channel": "general" }
```

### Authenticated request

Select a credential in the config panel. The credential's access token or API key is included automatically in the `Authorization` header.

## Response output

The HTTP Request node returns:

- `data` — parsed response body (JSON if applicable).
- `status` — HTTP status code.
- `headers` — response headers.

Downstream nodes access via the node's reference ID:

```
{{ http_request.data.results }}
{{ http_request.status }}
```
