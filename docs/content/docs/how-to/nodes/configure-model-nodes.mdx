---
title: Configure Model Nodes
description: Set up AI model nodes with OpenAI and Anthropic.
---

Model nodes send a prompt to an LLM and return the response. They support OpenAI (GPT-4o, GPT-4o-mini, etc.) and Anthropic (Claude Sonnet, Haiku, Opus).

## Add a Model node

1. Drag **Model** from the Core provider onto the canvas.
2. Connect an upstream node to provide input data.
3. Double-click to open the config panel.

## Configuration fields

| Field | Required | Description |
|-------|----------|-------------|
| **Credential** | Yes | An LLM API key credential (OpenAI or Anthropic). |
| **Model** | Yes | The model to use (e.g., `gpt-4o-mini`, `claude-sonnet-4-20250514`). |
| **Prompt** | Yes | The main prompt. Supports `{{ }}` template expressions. |
| **System Prompt** | No | Optional system instructions. |
| **Temperature** | No | Creativity (0.0–2.0). Lower = more deterministic. |
| **Max Tokens** | No | Maximum response length. |

## Using template expressions in prompts

The prompt field supports JavaScript template expressions. Reference upstream node data:

```
Summarise the following article for {{ input.audience }}:

{{ http_request.data.content }}
```

## Model output

The Model node outputs an object with:

- `content` — the LLM's text response.
- `usage` — token usage (`promptTokens`, `completionTokens`, `totalTokens`).
- `model` — the model used.

Downstream nodes access it via the Model node's reference ID:

```
{{ model.content }}
```
