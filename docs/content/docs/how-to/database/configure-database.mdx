---
title: Configure Database
description: Set up SQLite, PostgreSQL, or MySQL for Invect.
---

Invect supports three databases via Drizzle ORM:

| Database       | Best for                              | Driver                                 |
| -------------- | ------------------------------------- | -------------------------------------- |
| **SQLite**     | Local development, single-server apps | `better-sqlite3` or `libsql`           |
| **PostgreSQL** | Production, multi-server              | `postgres`, `pg`, or `neon-serverless` |
| **MySQL**      | Existing MySQL infrastructure         | `mysql2`                               |

## SQLite

The simplest setup — a single file database, no server required.

```bash
pnpm add better-sqlite3
```

```ts
database: {
  type: 'sqlite',
  connectionString: 'file:./dev.db',
}
```

For [Turso](https://turso.tech) or [libSQL](https://libsql.org):

```bash
pnpm add @libsql/client
```

```ts
database: {
  type: 'sqlite',
  connectionString: process.env.DATABASE_URL,
  driver: 'libsql',
}
```

## PostgreSQL

Recommended for production deployments.

```bash
pnpm add postgres
```

```ts
database: {
  type: 'postgresql',
  connectionString: 'postgresql://user:password@localhost:5432/invect',
}
```

For [Neon](https://neon.tech) serverless:

```bash
pnpm add @neondatabase/serverless
```

```ts
database: {
  type: 'postgresql',
  connectionString: process.env.DATABASE_URL,
  driver: 'neon-serverless',
}
```

## MySQL

```bash
pnpm add mysql2
```

```ts
database: {
  type: 'mysql',
  connectionString: 'mysql://user:password@localhost:3306/invect',
}
```
