## Selecting a credential in the config panel

When you add a node that requires authentication (e.g., a Gmail, Slack, or Model node), the config panel shows a **Credential** dropdown.

1. Double-click the node to open its config panel.
2. Find the **Credential** field.
3. Select a credential from the dropdown. Only credentials matching the node's provider are shown.
4. Close the config panel.

The credential is now attached to this specific node instance. When the node executes, it receives the decrypted credential (API key or OAuth2 access token).

## Which nodes need credentials?

- **Model nodes** (`core.model`) — require an LLM API key (OpenAI or Anthropic).
- **Agent nodes** — require an LLM API key for the underlying model.
- **Integration actions** (Gmail, Slack, GitHub, etc.) — require an OAuth2 credential for the corresponding provider.
- **HTTP Request** (`http.request`) — optionally accepts a credential for authenticated requests.
- **PostgreSQL** (`postgres.*`) — requires a database connection credential.

Core nodes like Input, Output, Template String, JavaScript, If/Else, and Switch do not require credentials.

## How credentials are resolved at runtime

When a node executes:

1. Invect looks up the credential ID stored in the node's config.
2. The credential is decrypted from the database.
3. For OAuth2 credentials, expired tokens are automatically refreshed.
4. The action's `execute` function receives the credential via `context.credential`.

Actions access the credential like this:

```ts
async execute(params, context) {
  const token = context.credential?.config?.accessToken;
  // or for API keys:
  const apiKey = context.credential?.config?.apiKey;
}
```

## Sharing credentials across flows

When creating a credential, toggle the **Shared** option to make it available to all flows and users. Unshared credentials are private to the user who created them.
