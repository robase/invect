---
title: Manage API Keys
description: Add, edit, and delete API key credentials for AI providers and services.
---

Credentials store sensitive keys and tokens that your flow nodes use to authenticate with external services. Invect encrypts all credentials at rest with AES-256-GCM.

## Add an API key credential

### Via the UI

1. Click **Credentials** in the sidebar.
2. Click **Add Credential**.
3. Fill in the form:
   - **Name**: A descriptive name (e.g., `OpenAI Production`).
   - **Type**: Select `llm` for AI providers, or `http-api` for other services.
   - **Provider**: Select the provider (e.g., `openai`, `anthropic`).
   - **Auth Type**: Select `API Key`.
   - **API Key**: Paste your key.
   - **Description** (optional): A note about this credential.
   - **Shared**: Toggle on if all users should be able to use this credential.
4. Click **Save**.

### Via the config file

You can seed credentials automatically on startup using `defaultCredentials` in your config:

```ts title="invect.config.ts"
import { defineConfig } from '@invect/core';

export const invectConfig = defineConfig({
  database: {
    type: 'sqlite',
    connectionString: 'file:./dev.db',
  },
  defaultCredentials: [
    {
      name: 'OpenAI',
      type: 'llm',
      provider: 'openai',
      authType: 'apiKey',
      config: { apiKey: process.env.OPENAI_API_KEY },
      description: 'OpenAI GPT models',
      isShared: true,
    },
    {
      name: 'Anthropic',
      type: 'llm',
      provider: 'anthropic',
      authType: 'apiKey',
      config: { apiKey: process.env.ANTHROPIC_API_KEY },
      description: 'Anthropic Claude models',
      isShared: true,
    },
  ],
});
```

Default credentials are created on startup if they don't already exist (matched by name). They are not updated if they already exist — edit them via the UI or API.

### Via the API

```bash
curl -X POST http://localhost:3000/invect/credentials \
  -H "Content-Type: application/json" \
  -d '{
    "name": "OpenAI",
    "type": "llm",
    "provider": "openai",
    "authType": "apiKey",
    "config": { "apiKey": "sk-..." },
    "description": "OpenAI GPT models",
    "isShared": true
  }'
```

## Edit a credential

1. Open the **Credentials** page.
2. Click the credential you want to edit.
3. Update the fields. If you need to rotate the API key, enter the new key in the **API Key** field.
4. Click **Save**.

Existing flow nodes that reference this credential will automatically use the updated key on the next execution.

## Delete a credential

1. Open the **Credentials** page.
2. Click the delete button on the credential.
3. Confirm deletion.

Any flow nodes that reference a deleted credential will fail at execution with a credential-not-found error.

## Encryption

All credential values are encrypted at rest using AES-256-GCM. In production, set the `INVECT_ENCRYPTION_KEY` environment variable to a 32-byte base64-encoded key:

```bash
# Generate a secure key
pnpm dlx @invect/cli secret
```

See [Encryption Keys](/docs/how-to/production/encryption-keys) for production setup.
