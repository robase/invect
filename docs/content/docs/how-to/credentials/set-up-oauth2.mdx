Invect supports OAuth2 authentication for 35 services including Google (Gmail, Drive, Docs, Sheets, Calendar), GitHub, Slack, Microsoft 365, Linear, Jira, and more.

## How OAuth2 works in Invect

1. You click "Connect" for a provider in the Invect UI.
2. A popup opens to the provider's authorization page.
3. You grant access and the popup closes.
4. Invect exchanges the authorization code for access and refresh tokens.
5. Tokens are encrypted and stored as a credential.
6. When a node uses the credential, Invect auto-refreshes expired tokens.

## Prerequisites

Before connecting an OAuth2 provider, you need **client credentials** (Client ID and Client Secret) from that provider's developer console. Here's where to get them for common providers:

- **Google**: [Google Cloud Console](https://console.cloud.google.com/apis/credentials)
- **GitHub**: [GitHub Developer Settings](https://github.com/settings/developers)
- **Slack**: [Slack API Apps](https://api.slack.com/apps)
- **Microsoft**: [Azure App Registrations](https://portal.azure.com/#blade/Microsoft_AAD_RegisteredApps)
- **Linear**: [Linear Settings > API](https://linear.app/settings/api)

See [OAuth2 Providers reference](/docs/reference/oauth2-providers) for the full list of supported providers.

## Connect a provider

1. Go to **Credentials** in the sidebar.
2. Click **Add Credential**.
3. Select **OAuth2** as the auth type.
4. Choose a provider (e.g., Google).
5. Enter your **Client ID** and **Client Secret**.
6. Select the **scopes** you need (e.g., Gmail read/write, Drive access).
7. Click **Connect**.
8. A popup opens — log in and authorize the app.
9. The popup closes and Invect stores the credential.

The credential now appears in your credentials list with the provider name and connected account.

## Set callback URLs

Your OAuth2 app registration needs a redirect URI. The default callback URL is:

```
http://localhost:3000/invect/credentials/oauth2/callback
```

For production, use your actual domain:

```
https://your-app.com/invect/credentials/oauth2/callback
```

If you're running the frontend separately (e.g., Vite dev server), the callback happens via `postMessage` from the popup to the parent window — the callback URL still points to your backend.

## Use OAuth2 credentials in flows

Once connected, OAuth2 credentials appear in the credential dropdown when configuring nodes that support that provider. For example:

- Select your **Google** credential when configuring a **Gmail: Send Message** action node.
- Select your **GitHub** credential for **GitHub: Create Issue**.
- Select your **Slack** credential for **Slack: Send Message**.

Invect automatically includes the `Authorization: Bearer <access_token>` header. If the token has expired and the provider supports refresh, Invect refreshes it transparently before the request.

## Token refresh

Most OAuth2 providers issue refresh tokens that let Invect get new access tokens without re-authorization. Invect handles this automatically:

1. Before using a credential, Invect checks the token's `expiresAt` timestamp.
2. If expired and a refresh token exists, Invect calls the provider's token endpoint.
3. The new access token (and optionally new refresh token) is saved.
4. The node executes with the fresh token.

A few providers (GitHub, Notion, Shopify) don't support refresh tokens. For these, you'll need to re-authorize when the token expires.
