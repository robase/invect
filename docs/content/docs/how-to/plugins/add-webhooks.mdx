---

The `@invect/webhooks` plugin adds webhook trigger support with HMAC signature verification, rate limiting, and deduplication.

## Install

```bash
pnpm add @invect/webhooks
```

## Configure

```ts title="invect.config.ts"
import { defineConfig } from '@invect/core';
import { webhooksPlugin } from '@invect/webhooks';

export const invectConfig = defineConfig({
  database: {
    type: 'sqlite',
    connectionString: 'file:./dev.db',
  },
  plugins: [
    webhooksPlugin({
      webhookBaseUrl: process.env.WEBHOOK_BASE_URL || 'http://localhost:3000/invect',
      rateLimitMaxRequests: 60,     // Max requests per window (default: 60)
      rateLimitWindowMs: 60_000,    // Rate limit window in ms (default: 60s)
      dedupTtlMs: 86_400_000,      // Deduplication TTL in ms (default: 24h)
    }),
  ],
});
```

Regenerate schemas:

```bash
pnpm dlx @invect/cli generate
pnpm dlx @invect/cli migrate --push
```

## Create a webhook trigger

1. Open a flow in the editor.
2. Add a **Webhook Trigger** node from the Triggers provider.
3. Configure the allowed HTTP methods and optional HMAC secret.
4. The webhook URL is displayed in the config panel.

## Send a webhook

```bash
curl -X POST https://your-app.com/invect/webhooks/{webhookId} \
  -H "Content-Type: application/json" \
  -d '{ "event": "order.created", "data": { "id": 123 } }'
```

The webhook payload is passed as input to the flow.

## Security features

- **HMAC signature verification** — configure a shared secret per webhook. Invect validates the `X-Webhook-Signature` header.
- **Rate limiting** — per-webhook rate limits prevent abuse.
- **Deduplication** — repeated payloads within the TTL are ignored.

## Frontend setup

```tsx
import { webhooksFrontendPlugin } from '@invect/webhooks/ui';

// Add to your plugins array:
plugins={[authFrontendPlugin, rbacFrontendPlugin, webhooksFrontendPlugin]}
```
