title: Use Triggers

Triggers define how a flow starts. Invect provides two built-in trigger types: **Manual** and **Cron**. Webhook triggers are available via the [Webhooks plugin](/docs/how-to/plugins/add-webhooks).

## Manual trigger

A manual trigger means the flow is started explicitly — via the UI "Run" button, the REST API, or programmatic code. This is the default for flows without a trigger node.

To add a manual trigger node:

1. Drag the **Manual Trigger** from the Triggers provider onto the canvas.
2. Connect it as the first node in your flow (before Input or other nodes).

Manual trigger nodes are optional — flows without them can still be run via the UI or API.

## Cron trigger

A cron trigger runs the flow automatically on a schedule.

1. Drag a **Cron Trigger** from the Triggers provider onto the canvas.
2. Double-click to open the config panel.
3. Set the **cron expression** (e.g., `0 9 * * *` for 9 AM daily).
4. Optionally provide **static inputs** that are passed to the flow on each run.

### Cron expression syntax

```
┌───────────── minute (0–59)
│ ┌───────────── hour (0–23)
│ │ ┌───────────── day of month (1–31)
│ │ │ ┌───────────── month (1–12)
│ │ │ │ ┌───────────── day of week (0–7, Sun=0 or 7)
│ │ │ │ │
* * * * *
```

### Common schedules

| Expression | Schedule |
|------------|----------|
| `* * * * *` | Every minute |
| `0 * * * *` | Every hour |
| `0 9 * * *` | 9:00 AM daily |
| `0 9 * * 1-5` | 9:00 AM weekdays |
| `0 0 * * *` | Midnight daily |
| `0 0 1 * *` | First of month |

### Disabling cron triggers

To disable cron globally (e.g., in test environments), set `cronEnabled: false` in your config:

```ts
triggers: {
  cronEnabled: false,
}
```

## Webhook triggers

For webhook-based flow triggers, install the [Webhooks plugin](/docs/how-to/plugins/add-webhooks). It provides:

- Per-flow webhook URLs
- HMAC signature verification
- Rate limiting and deduplication
