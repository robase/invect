## Create a flow

### Via the UI

1. Open the dashboard and click **Create Flow**.
2. Enter a name and optional description.
3. Click **Create** — the flow editor opens.

### Via the API

```bash
curl -X POST http://localhost:3000/invect/flows \
  -H "Content-Type: application/json" \
  -d '{ "name": "My Flow", "description": "A workflow" }'
```

## Edit a flow

Open a flow from the dashboard to enter the visual editor. Changes are saved automatically as you:

- Add or remove nodes from the palette.
- Connect nodes by dragging between handles.
- Configure nodes by double-clicking to open the config panel.
- Reposition nodes on the canvas.

## Flow versions

Each time you save meaningful changes, Invect creates a new version of the flow's definition (its nodes, edges, and positions). You can view version history and restore previous versions.

Versions track the flow's **definition** — the graph structure and node configs. Execution history (flow runs) is stored separately and linked to the version that was active when the run started.

## Duplicate a flow

To duplicate an existing flow, use the flow menu (three dots) on the dashboard card and select **Duplicate**. This creates a new flow with the same definition but a new ID.

## Delete a flow

From the dashboard, click the flow menu and select **Delete**. This removes the flow definition and all its versions. Execution history for the flow is preserved.

## Run a flow

Click the **Run** button in the toolbar. If the flow has an Input node, a dialog prompts for input values. Click **Run** to execute.

You can also run individual nodes: right-click a node and select **Run Node**. This executes only that node with its current input data (mocked or from a previous run).
