---
title: Set Up with Next.js
description: Install and configure Invect in a Next.js App Router application.
---

## Install packages

```bash
pnpm add @invect/core @invect/nextjs @invect/ui better-sqlite3
```

For PostgreSQL, replace `better-sqlite3` with `postgres`.

## Generate and migrate the database

Create `invect.config.ts` at the project root:

```ts title="invect.config.ts"
import { defineConfig } from '@invect/core';

export default defineConfig({
  database: {
    type: 'sqlite',
    connectionString: 'file:./invect.db',
  },
});
```

Run:

```bash
pnpm dlx @invect/cli generate
pnpm dlx @invect/cli migrate --push
```

## Create the API route

Create a catch-all API route for Invect:

```ts title="app/api/invect/[...invect]/route.ts"
import { createInvectHandler } from '@invect/nextjs';

const handler = createInvectHandler({
  database: {
    type: 'sqlite',
    connectionString: 'file:./invect.db',
  },
  logging: {
    level: 'info',
  },
});

export {
  handler as GET,
  handler as POST,
  handler as PUT,
  handler as DELETE,
  handler as PATCH,
};
```

`createInvectHandler` returns a single handler function that you export for all HTTP methods. The catch-all route `[...invect]` matches all Invect API paths.

## Mount the visual editor

Create a client page for the Invect UI:

```tsx title="app/invect/[[...slug]]/page.tsx"
'use client';

import dynamic from 'next/dynamic';
import '@invect/ui/styles';

const Invect = dynamic(
  () => import('@invect/ui').then((mod) => ({ default: mod.Invect })),
  { ssr: false }
);

export default function InvectPage() {
  return (
    <div style={{ height: '100vh' }}>
      <Invect apiBaseUrl="/api/invect" basePath="/invect" />
    </div>
  );
}
```

The `dynamic` import with `ssr: false` is required because the flow editor uses browser-only APIs (React Flow, drag-and-drop). The `[[...slug]]` catch-all route lets the Invect UI handle its own client-side routing.

## Start the dev server

```bash
pnpm dev
```

Open `http://localhost:3000/invect` to see the flow editor.

## Add plugins (optional)

For authentication with the Next.js adapter:

```bash
pnpm add @invect/user-auth @invect/rbac
```

Update your API route:

```ts title="app/api/invect/[...invect]/route.ts"
import { createInvectHandler } from '@invect/nextjs';
import { userAuth } from '@invect/user-auth';
import { rbacPlugin } from '@invect/rbac';

const handler = createInvectHandler({
  database: {
    type: 'sqlite',
    connectionString: 'file:./invect.db',
  },
  plugins: [
    userAuth({
      betterAuthOptions: { secret: process.env.BETTER_AUTH_SECRET },
      globalAdmins: [{ email: 'admin@example.com', pw: 'changeme', name: 'Admin' }],
    }),
    rbacPlugin(),
  ],
});

export {
  handler as GET,
  handler as POST,
  handler as PUT,
  handler as DELETE,
  handler as PATCH,
};
```

Re-run `pnpm dlx @invect/cli generate && pnpm dlx @invect/cli migrate --push` after adding plugins that require database tables.
