---
title: Add Custom Tools
description: Register custom tools for agent nodes.
---

You can add custom tools for agents in two ways: as actions (which also become flow nodes) or as standalone tools (agent-only).

## Option A: Create an action (recommended)

Actions created with `defineAction()` automatically register as both flow nodes and agent tools. See [Create Custom Actions](/docs/how-to/nodes/create-custom-actions).

This is the preferred approach because:
- The tool is also usable as a flow node.
- It gets Zod validation, UI config fields, and credential support.
- It appears in both the node palette and the agent tool selector.

## Option B: Create a standalone tool

For utility tools that don't make sense as flow nodes:

```ts title="my-tool.ts"
import type { AgentToolDefinition, AgentToolExecutor } from '@invect/core';

export const myToolDefinition: AgentToolDefinition = {
  id: 'my_custom_tool',
  name: 'My Custom Tool',
  description: 'Describe what this tool does — the LLM uses this to decide when to call it.',
  inputSchema: {
    type: 'object',
    properties: {
      query: { type: 'string', description: 'The search query' },
    },
    required: ['query'],
  },
  category: 'utility',
  enabledByDefault: false,
};

export const myToolExecutor: AgentToolExecutor = async (input, context) => {
  const result = await someOperation(input.query);
  return { success: true, output: result };
};
```

Register the tool in a plugin:

```ts title="invect.config.ts"
import { defineConfig } from '@invect/core';

export const invectConfig = defineConfig({
  database: { type: 'sqlite', connectionString: 'file:./dev.db' },
  plugins: [{
    id: 'my-tools',
    init: async (ctx) => {
      ctx.registerTool(myToolDefinition, myToolExecutor);
    },
  }],
});
```

## Built-in standalone tools

Invect includes two standalone agent tools:

| Tool | Description |
|------|-------------|
| `math_eval` | Evaluate mathematical expressions. |
| `json_logic` | Apply JSON Logic rules to data. |
