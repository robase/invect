---
title: Triggers
description: Run flows on a schedule or in response to events.
---

Flows can be triggered manually (the Run button or an API call), on a schedule (cron), or by external events (webhooks). Triggers are configured per-flow and can be combined.

## Manual triggers

Every flow supports manual execution out of the box. In the editor, click **Run** and provide input values. Via the API:

```bash
curl -X POST http://localhost:3000/invect/flows/:flowId/run \
  -H "Content-Type: application/json" \
  -d '{"inputs": {"name": "Alice"}}'
```

There's also a **Trigger: Manual** node you can add to a flow to make the trigger explicit in the graph. It doesn't change behaviour — it's a visual marker indicating the flow is designed for on-demand use.

## Cron triggers

Cron triggers run flows on a schedule. Add a **Trigger: Cron** node to your flow and configure the cron expression in its config panel.

The expression follows standard cron syntax with five fields:

```
┌────── minute (0-59)
│ ┌──── hour (0-23)
│ │ ┌── day of month (1-31)
│ │ │ ┌ month (1-12)
│ │ │ │ ┌ day of week (0-7, 0 and 7 = Sunday)
│ │ │ │ │
* * * * *
```

Common patterns:

| Expression     | Schedule                       |
| -------------- | ------------------------------ |
| `0 9 * * *`    | Daily at 9:00 AM               |
| `*/15 * * * *` | Every 15 minutes               |
| `0 0 * * 1`    | Every Monday at midnight       |
| `0 8 1 * *`    | First of each month at 8:00 AM |

Cron triggers are managed by the batch polling service, which checks for due triggers on each tick. The `execution.staleRunCheckIntervalMs` config option controls the polling interval.

Cron-triggered flows don't receive user input — they run with empty inputs (or whatever the trigger node produces).

## Webhook triggers

With the `@invect/webhooks` plugin, flows can be triggered by inbound HTTP requests. Each webhook has a unique URL, and the request body becomes the flow's input data.

```ts
import { webhooksPlugin } from '@invect/webhooks';

const config = defineConfig({
  // ...
  plugins: [webhooksPlugin()],
});
```

See [Webhooks plugin](/docs/plugins/webhooks) for configuration details.

## Trigger nodes vs. API triggering

Trigger nodes (Manual, Cron, Webhook) are visual elements in the flow graph. They sit at the start of a flow and represent how the flow is intended to be started. But any flow can also be triggered via the REST API regardless of what trigger nodes it contains — the trigger nodes are informational, not restrictive.
