Integration nodes connect your flows to third-party services. Each action handles authentication, request formatting, and response parsing — you just configure the parameters.

Most integration nodes require an [OAuth2 credential](/docs/credentials/oauth2) or [API key](/docs/credentials/api-keys).

## Google Workspace

### Gmail

| Action ID             | Description                    |
| --------------------- | ------------------------------ |
| `gmail.list_messages` | List messages matching a query |
| `gmail.get_message`   | Get a single message by ID     |
| `gmail.send_message`  | Send an email                  |

**Credential:** Google (Gmail) OAuth2

## Adding custom integrations

You can create custom action nodes for any service. See [Concepts > Actions System](/docs/concepts/actions-system) for how to define new actions, or use the **HTTP Request** node (`http.request`) for quick one-off integrations.
