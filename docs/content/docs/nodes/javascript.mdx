---
title: JavaScript Node
description: Run custom JavaScript in a sandboxed QuickJS environment.
---

The JavaScript node runs arbitrary code in a QuickJS WASM sandbox. It's the escape hatch for transformations too complex for templates or JQ ŌĆö custom logic, multi-step calculations, or data reshaping that needs imperative code.

## How it works

The node receives the incoming data object (all upstream outputs keyed by reference ID) and executes your script. The script's return value becomes the node's output.

```js
// incoming data: { "api_response": { "users": [...] } }

const users = api_response.users;
const active = users.filter(u => u.active);
return {
  total: users.length,
  active: active.length,
  names: active.map(u => u.name),
};
```

All upstream node outputs are available as top-level variables ŌĆö you don't need to access them through a namespace. The variable names match the upstream nodes' reference IDs.

## The sandbox

Code runs in QuickJS compiled to WASM ŌĆö not in Node.js. This means:

**Available:** Standard JavaScript (ES2020), `JSON`, `Math`, `Date`, `Array`/`Object`/`String` methods, `RegExp`, `Map`, `Set`, `parseInt`/`parseFloat`, `encodeURIComponent`/`decodeURIComponent`, `atob`/`btoa`.

**Not available:** Node.js APIs (`fs`, `http`, `process`, `Buffer`), `fetch`, `setTimeout`/`setInterval`, `require`/`import`, `console.log` (use `return` to output values).

The sandbox isolates execution ŌĆö scripts can't access the file system, make network requests, or affect the host process. This is by design: the JavaScript node is for data transformation, not side effects. For network calls, use the HTTP Request node.

## Return values

Whatever you `return` from the script becomes the node's output. This can be:

- A primitive (string, number, boolean)
- An object
- An array
- `null` or `undefined`

If you don't explicitly `return`, the node outputs `undefined`.

## Error handling

If the script throws an error, the node fails and the error message is recorded in the execution trace. Downstream nodes are skipped. You can use try/catch within the script to handle expected errors gracefully:

```js
try {
  const parsed = JSON.parse(raw_data.body);
  return { success: true, data: parsed };
} catch (e) {
  return { success: false, error: "Invalid JSON" };
}
```

## When to use JavaScript vs. other nodes

- **Templates** ŌĆö simple data access, string composition, basic expressions
- **JQ** ŌĆö structured JSON transformations (filtering, mapping, reshaping)
- **JavaScript** ŌĆö complex logic, multiple steps, conditionals, loops, custom algorithms
- **HTTP Request** ŌĆö external API calls (JavaScript can't make network requests)
