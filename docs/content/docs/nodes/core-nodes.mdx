## Input

**Action ID:** `core.input`

Defines the inputs your flow expects. Add named fields with types and default values. When the flow runs, input values are passed from the trigger or API call.

| Param              | Type  | Description                                                   |
| ------------------ | ----- | ------------------------------------------------------------- |
| `inputDefinitions` | array | Named inputs with `name`, `type`, and optional `defaultValue` |

Downstream nodes reference inputs via `{{ input.fieldName }}`.

## Output

**Action ID:** `core.output`

Captures the final result of a flow. Place at the end of your flow to define what the flow returns.

The Output node automatically receives all upstream node data. You can use template expressions to select specific fields for the output.

## Model

**Action ID:** `core.model`

Calls an LLM (OpenAI, Anthropic, or OpenRouter) with a prompt and returns the response.

| Param            | Type       | Description                                                  |
| ---------------- | ---------- | ------------------------------------------------------------ |
| `credentialId`   | credential | API key for the AI provider                                  |
| `model`          | select     | Model name (e.g., `gpt-4o-mini`, `claude-sonnet-4-20250514`) |
| `prompt`         | textarea   | The prompt to send (supports template expressions)           |
| `systemPrompt`   | textarea   | Optional system instructions                                 |
| `temperature`    | number     | Sampling temperature (0–2)                                   |
| `maxTokens`      | number     | Maximum response tokens                                      |
| `responseFormat` | select     | `text` or `json_object`                                      |

### Example

```
Prompt: Summarize the following article in 3 bullet points:

{{ fetch_article.content }}
```

<Callout type="info">
  The Model node makes a single API call. For iterative multi-step reasoning with tool calling, use
  the [Agent](/docs/agents) node instead.
</Callout>

## Template String

**Action ID:** `core.template_string`

Generates a text string using template expressions. Useful for building prompts, formatting data, or constructing messages.

| Param      | Type     | Description                   |
| ---------- | -------- | ----------------------------- |
| `template` | textarea | Text with `{{ }}` expressions |

### Example

```
Report for {{ input.date }}:
- Total users: {{ analytics.totalUsers }}
- Active: {{ analytics.activeUsers }}
- Revenue: ${{ analytics.revenue.toFixed(2) }}
```

## If/Else

**Action ID:** `core.if_else`

Evaluates a JavaScript condition and routes execution to the `true` or `false` branch.

| Param       | Type | Description                               |
| ----------- | ---- | ----------------------------------------- |
| `condition` | code | JavaScript expression returning a boolean |

The If/Else node has **two output handles** — connect downstream nodes to either the `true` or `false` output. Nodes on the inactive branch are automatically skipped.

The node acts as a **passthrough** — its output equals its input data. This means downstream nodes on the active branch receive the same data the If/Else node received.

### Example conditions

```js
// Check a value
{
  {
    input.score > 80;
  }
}

// String comparison
{
  {
    api_result.status === 'success';
  }
}

// Array check
{
  {
    input.items.length > 0;
  }
}

// Complex logic
{
  {
    input.age >= 18 && input.country === 'US';
  }
}
```

## JQ

**Action ID:** `core.jq`

Transforms JSON data using [JQ](https://jqlang.github.io/jq/) query syntax. Powerful for selecting, filtering, and reshaping data.

| Param   | Type | Description     |
| ------- | ---- | --------------- |
| `query` | code | JQ query string |

### Example queries

```text
# Select a field
.users[0].name

# Filter array
.items | map(select(.active == true))

# Reshape data
{ names: [.users[].name], count: (.users | length) }
```

The JQ node operates on the incoming data object — all upstream node outputs are available.

## HTTP Request

**Action ID:** `http.request`

Makes HTTP requests to any URL. Supports all methods, custom headers, JSON/form bodies, and authentication.

| Param          | Type          | Description                                 |
| -------------- | ------------- | ------------------------------------------- |
| `url`          | text          | Request URL (supports template expressions) |
| `method`       | select        | `GET`, `POST`, `PUT`, `PATCH`, `DELETE`     |
| `headers`      | json          | Request headers as key-value pairs          |
| `body`         | json/textarea | Request body                                |
| `credentialId` | credential    | Optional credential for authentication      |

### Example

```
URL: https://api.example.com/users/{{ input.userId }}
Method: GET
Headers: { "Authorization": "Bearer {{ api_key.token }}" }
```

The response (parsed JSON or raw text) is available to downstream nodes.

## Agent

**Type:** `AGENT`

An AI agent that runs an iterative prompt → tool → response loop. Unlike the Model node (single call), the Agent can use tools, analyze results, and make multiple calls until it reaches a conclusion.

See [Agents](/docs/agents) for full documentation.
