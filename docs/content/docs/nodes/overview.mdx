Nodes are the building blocks of flows. Each node performs a specific action — calling an AI model, making an HTTP request, transforming data, or interacting with a third-party service.

## How nodes work

Every node is defined as an **action** using the `defineAction()` pattern. An action is a self-contained definition that specifies:

- **ID** — Unique identifier (e.g., `core.model`, `gmail.send_message`)
- **Provider** — Grouping for the UI palette (e.g., Core, Gmail, Slack)
- **Params** — Zod-validated configuration schema with UI field definitions
- **Execute function** — The actual logic that runs when the node executes

When a node executes, Invect:

1. Builds the **incoming data object** from upstream node outputs
2. **Resolves template expressions** in all config params
3. **Validates params** against the Zod schema
4. Calls the **execute function** with validated params and context
5. Stores the **output** for downstream nodes

## Node categories

### Core nodes

Built-in nodes for flow control, data transformation, and AI:

| Node | Action ID | Description |
|------|-----------|-------------|
| Input | `core.input` | Define flow inputs with types and defaults |
| Output | `core.output` | Capture the final result of a flow |
| Model | `core.model` | Call OpenAI, Anthropic, or OpenRouter models |
| Template String | `core.template_string` | Generate text with template expressions |
| If/Else | `core.if_else` | Branch based on a JavaScript condition |
| JQ | `core.jq` | Transform JSON with JQ queries |
| HTTP Request | `http.request` | Make HTTP requests to any URL |
| Agent | `AGENT` | AI agent with iterative tool calling |

See [Core Nodes](/docs/nodes/core-nodes) for detailed configuration of each.

## Node configuration

Every node has a config panel in the visual editor. Fields are defined by the action's `params.fields` array. Common field types:

| Field Type | Description |
|------------|-------------|
| `text` | Single-line text input |
| `textarea` | Multi-line text input |
| `code` | Code editor (JQ, SQL, JavaScript) |
| `select` | Dropdown selection |
| `number` | Numeric input |
| `boolean` | Toggle switch |
| `credential` | Credential selector |
| `json` | JSON editor |

All text-based fields support [template expressions](/docs/flows/templates) — use `{{ }}` to reference upstream node outputs.

## Node outputs

Every node produces an output that downstream nodes can reference:

```ts
{
  success: true,
  output: { /* action-specific result */ }
}
```

Some nodes (like If/Else) produce **multiple output ports** with named variables:

```ts
{
  success: true,
  output: { /* passthrough data */ },
  outputVariables: {
    true_output: { /* data for true branch */ },
    false_output: { /* data for false branch */ }
  }
}
```

## Credentials

Many integration nodes require credentials (OAuth2 tokens or API keys). Attach a credential to a node in its config panel. See [Credentials](/docs/credentials) for setup.

Nodes that need credentials declare this in their action definition:

```ts
credential: {
  required: true,
  oauth2Provider: 'google_docs',
  description: 'Google Docs OAuth2 credential',
}
```
