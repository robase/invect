Every node in a flow is backed by an **action** — a self-contained definition that includes configuration schema, UI fields, and execution logic. The node palette in the visual editor is populated from the action registry.

## Core nodes

These are the fundamental building blocks, available without any external credentials.

### Input

Defines the flow's input schema. Configure named fields with types (`string`, `number`, `boolean`, `json`). When the flow runs, the Input node captures the provided values and makes them available to downstream nodes via its reference ID.

### Output

Defines what the flow returns. Connect it at the end of your pipeline — it captures whatever its upstream node produces as the flow's result. No configuration needed.

### Model

Sends a prompt to an LLM (OpenAI, Anthropic, or OpenRouter). Requires a credential. The prompt supports `{{ }}` template expressions. Returns the model's response text, token usage, and metadata. Can be configured for batch mode. See [Model nodes](/docs/nodes/model-nodes).

### Template String

Composes a string from a template using `{{ }}` expressions. Useful for formatting data between nodes, building prompts, or constructing messages. See [Templates and data flow](/docs/flows/templates-and-data).

### JavaScript

Runs arbitrary JavaScript in a QuickJS WASM sandbox. Takes incoming data as input and returns the script's result. Good for data transformations too complex for templates. See [JavaScript node](/docs/nodes/javascript).

### JQ

Applies a JQ query to the incoming data. Powerful for extracting and reshaping JSON — more concise than JavaScript for structured data operations.

### If/Else

Evaluates a condition and routes execution to one of two branches. See [Conditional logic](/docs/flows/conditional-logic).

### Switch

Routes execution to one of multiple named branches based on a value. Like if/else but with more than two outcomes.

## HTTP

The **HTTP Request** node makes outbound HTTP calls. Configure the method, URL, headers, and body — all supporting template expressions. See [HTTP requests](/docs/nodes/http-requests).

## Trigger nodes

- **Manual** — visual marker for on-demand flows
- **Cron** — runs the flow on a schedule
- **Webhook** — runs the flow in response to inbound HTTP requests (requires the webhooks plugin)

See [Triggers](/docs/flows/triggers).

## Agent

The Agent node is a special case — it runs an LLM in an iterative loop with access to tools. Unlike other nodes, agents can make multiple decisions and tool calls before producing a final result. See [Agents](/docs/agents/overview).

## Integration nodes

190+ actions across 38+ providers. Each integration action works both as a flow node and as an agent tool. See the [full list](/docs/nodes/built-in-actions).

Examples: Gmail (send, list, read), Slack (send message, list channels), GitHub (create issue, list repos), Google Sheets (read, write, append), Salesforce, Jira, Linear, Stripe, and many more.
