In this tutorial you will add an AI Model node to the greeting flow from the previous tutorial. By the end, your flow will take a user's name and topic of interest, then use an LLM to generate a personalised welcome message.

## Prerequisites

- Complete the [Build Your First Flow](/docs/tutorials/build-your-first-flow) tutorial.
- Have an **OpenAI** or **Anthropic** API key ready.

## Add an API key credential

Before you can use AI models, you need to store your API key as a credential in Invect.

1. In the Invect dashboard, click **Credentials** in the sidebar.
2. Click **Add Credential**.
3. Fill in the form:
   - **Name**: `OpenAI` (or `Anthropic`)
   - **Type**: `llm`
   - **Provider**: `openai` (or `anthropic`)
   - **Auth Type**: `API Key`
   - **API Key**: paste your key
4. Click **Save**.

Your API key is now stored with AES-256-GCM encryption and ready to use in flows.

## Update the Input node

Open the **Greeting Flow** from the previous tutorial. Double-click the Input node and add a second field:

- **Name**: `topic`
- **Type**: `string`

The flow will now accept both `name` and `topic` as inputs.

## Add a Model node

1. Open the node palette and find **Model** under the **Core** provider.
2. Drag it onto the canvas between the Input node and the Template String node.
3. Delete the existing edge between Input and Template String (click the edge, then press Delete).
4. Connect: **Input** â†’ **Model** â†’ **Template String**.
5. Double-click the Model node to open its config panel.

Configure the Model node:

- **Credential**: select the API key credential you created.
- **Model**: pick a model (e.g., `gpt-4o-mini` for OpenAI or `claude-sonnet-4-20250514` for Anthropic).
- **Prompt**:
  ```
  Write a friendly, one-paragraph welcome message for {{ input.name }} who is interested in {{ input.topic }}. Keep it concise and enthusiastic.
  ```

The prompt uses the same `{{ }}` template syntax as the Template String node. It references the upstream Input node's fields directly.

## Update the Template String

Now that the Model node sits between Input and the Template String, the template needs to reference the Model node's output instead:

```
{{ model.content }}
```

The key `model` is the Model node's reference ID (visible in its config panel). The `.content` field contains the LLM's text response.

Alternatively, you can delete the Template String and Output nodes entirely and just use a single Output node connected to the Model node â€” the Model node's response will pass through directly.

## Run the flow

1. Click **Run**.
2. Enter inputs:
   - **name**: `Alice`
   - **topic**: `machine learning`
3. Click **Run**.

Watch the flow execute:

- The **Input** node captures both values.
- The **Model** node sends the prompt to the LLM API and streams back a response.
- The **Output** node returns the AI-generated welcome message.

Click the Model node to inspect its execution â€” you'll see the full prompt that was sent, the model response, token usage, and latency.
