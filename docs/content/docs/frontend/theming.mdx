## Theme modes

Invect supports three theme modes: `light`, `dark`, and `system`. Set the theme on the `<Invect />` or `<InvectShell />` component.

The `system` mode automatically follows the operating system's preference and updates in real-time.

The theme choice is persisted to `localStorage` under the `invect-ui-theme` key.

## CSS scoping

All Invect styles are scoped inside a `.invect` CSS class. This means:

- Invect styles won't leak into your app
- Your app styles won't affect the Invect UI
- Dark mode is applied via `.invect.dark` class

```html
<!-- Rendering structure -->
<div class="invect dark">
  <!-- All Invect UI lives here -->
  <!-- imp-* theme tokens are active -->
</div>
```

## Theme tokens

Invect uses CSS custom properties with the `imp-` prefix. These control all colors, borders, shadows, and typography:

| Token                      | Description                 |
| -------------------------- | --------------------------- |
| `--imp-background`         | Page background             |
| `--imp-foreground`         | Default text color          |
| `--imp-card`               | Card/panel background       |
| `--imp-card-foreground`    | Card text color             |
| `--imp-popover`            | Popover/dropdown background |
| `--imp-primary`            | Primary accent color        |
| `--imp-primary-foreground` | Text on primary color       |
| `--imp-secondary`          | Secondary background        |
| `--imp-muted`              | Muted/disabled background   |
| `--imp-muted-foreground`   | Muted text color            |
| `--imp-accent`             | Accent highlight            |
| `--imp-destructive`        | Error/danger color          |
| `--imp-border`             | Border color                |
| `--imp-input`              | Input field border          |
| `--imp-ring`               | Focus ring color            |
| `--imp-radius`             | Border radius               |

## Using tokens in plugin components

If you're building a [frontend plugin](/docs/plugins/building-plugins), use Tailwind classes with the `imp-` token prefix:

```tsx
function MyPluginComponent() {
  return (
    <div className="bg-imp-background text-imp-foreground border border-imp-border rounded-md p-4">
      <h2 className="text-imp-primary font-semibold">Plugin Panel</h2>
      <p className="text-imp-muted-foreground">This component uses Invect theme tokens.</p>
    </div>
  );
}
```

These classes only work inside the `.invect` CSS scope (provided by `<Invect />` or `<InvectShell />`).

## Dark mode

Dark mode is handled automatically. Components should use `imp-*` tokens exclusively — never hardcode colors. When the theme switches between light and dark, token values update via CSS and all components re-render with the new palette.

```tsx
// CORRECT — adapts to light/dark
<div className="bg-imp-background text-imp-foreground" />

// WRONG — ignores theme
<div className="bg-white text-black" />
```
