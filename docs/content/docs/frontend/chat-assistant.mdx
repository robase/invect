The chat assistant is an AI-powered interface for building and editing flows through natural language. Instead of dragging nodes manually, you describe what you want and the assistant creates or modifies the flow for you.

## Enabling the chat

The chat assistant requires an LLM credential (OpenAI or Anthropic). Once a credential is available, the chat panel appears in the flow editor's sidebar.

Check whether chat is enabled:

```
GET /chat/status
```

Returns `{ enabled: true, models: [...] }` if at least one LLM credential is configured.

## How it works

The chat uses your configured LLM credential to process natural language requests. It has context about the current flow's structure (nodes, edges, configuration) and can:

- Add, remove, or reconfigure nodes
- Create or modify edges between nodes
- Suggest flow structures based on a description
- Explain what existing nodes do

The chat maintains message history per flow, so you can have an ongoing conversation about the flow you're building.

## Message persistence

Chat messages are saved per flow:

- `GET /chat/messages/:flowId` — retrieve saved messages
- `PUT /chat/messages/:flowId` — save messages
- `DELETE /chat/messages/:flowId` — clear message history

This lets you pick up conversations where you left off.

## Selecting a model

The chat model can be configured per session. Available models depend on your LLM credentials:

```
GET /chat/models/:credentialId?q=gpt
```

Returns models matching the query for the given credential.
