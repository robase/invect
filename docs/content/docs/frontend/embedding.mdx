The `@invect/ui` package provides a React component that renders the full visual flow editor, dashboard, and management UI.

## Installation

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tab value="pnpm">```bash pnpm add @invect/ui ```</Tab>
  <Tab value="npm">```bash npm install @invect/ui ```</Tab>
  <Tab value="yarn">```bash yarn add @invect/ui ```</Tab>
</Tabs>

## Basic usage

```tsx title="App.tsx"
import { Invect } from '@invect/ui';
import '@invect/ui/styles';

function App() {
  return <Invect apiBaseUrl="http://localhost:3000/invect" />;
}
```

This renders the full Invect UI including:

- Dashboard with flow cards
- Visual flow editor with drag-and-drop
- Node config panels
- Execution trace viewer
- Credential management
- Agent tool selector

## Props

| Prop               | Type                     | Default      | Description                                            |
| ------------------ | ------------------------ | ------------ | ------------------------------------------------------ |
| `apiBaseUrl`       | `string`                 | `undefined`  | Base URL of the Invect API backend                     |
| `basePath`         | `string`                 | `undefined`  | URL path where the UI is mounted (for correct routing) |
| `useMemoryRouter`  | `boolean`                | `false`      | Use in-memory router instead of browser URL routing    |
| `reactQueryClient` | `QueryClient`            | auto-created | Custom TanStack Query client                           |
| `plugins`          | `InvectFrontendPlugin[]` | `[]`         | Frontend plugins for sidebar items, routes, etc.       |

## Routing

The `<Invect />` component uses React Router v7 internally. It needs to know:

1. **Where the API is** → `apiBaseUrl`
2. **Where the UI is mounted** → `basePath`

### Standalone app

If Invect is the only thing on the page, you don't need `basePath`:

```tsx
<Invect apiBaseUrl="http://localhost:3000/invect" />
```

### Mounted at a sub-path

If Invect is at `/invect` in your app:

```tsx
<Invect apiBaseUrl="/api/invect" basePath="/invect" />
```

### Embedded in another app

If you're embedding Invect inside an existing React app with its own router, use `useMemoryRouter` to avoid conflicts:

```tsx
<Invect apiBaseUrl="/api/invect" useMemoryRouter />
```

## Two components: `Invect` vs `InvectShell`

| Component     | Purpose                                                        |
| ------------- | -------------------------------------------------------------- |
| `Invect`      | Full app — router, sidebar, pages, API providers, query client |
| `InvectShell` | CSS scope only — theme tokens, dark mode, no app logic         |

`InvectShell` exists for situations where you need Invect styling but not the full app — typically for plugin UI that renders **outside** the main component (like a sign-in page).

```tsx title="Using InvectShell for a login page"
import { InvectShell } from '@invect/ui';
import '@invect/ui/styles';

function LoginPage() {
  return (
    <InvectShell theme="dark">
      {/* Your login form gets Invect theme tokens */}
      <div className="bg-imp-background text-imp-foreground">
        <h1>Sign In</h1>
      </div>
    </InvectShell>
  );
}
```

### InvectShell props

| Prop        | Type                            | Default     | Description            |
| ----------- | ------------------------------- | ----------- | ---------------------- |
| `children`  | `ReactNode`                     | —           | Content to wrap        |
| `theme`     | `'light' \| 'dark' \| 'system'` | `'dark'`    | Color theme            |
| `className` | `string`                        | `undefined` | Additional CSS classes |

## With authentication

When using the [Authentication Plugin](/docs/plugins/authentication), use `AuthenticatedInvect` to wrap the UI with a sign-in gate:

```tsx title="App.tsx"
import { Invect, InvectShell } from '@invect/ui';
import { AuthenticatedInvect } from '@invect/user-auth/ui';
import '@invect/ui/styles';

function App() {
  return (
    <AuthenticatedInvect
      apiBaseUrl="/api/invect"
      basePath="/invect"
      InvectComponent={Invect}
      ShellComponent={InvectShell}
      theme="dark"
    />
  );
}
```

This renders:

```
InvectShell (CSS scope + theme)
  └─ AuthProvider
      └─ AuthGate
          ├─ Not authenticated → Sign-in page (with Invect styling)
          └─ Authenticated → <Invect /> (full app)
```

See [Authentication Plugin](/docs/plugins/authentication) for backend setup.

## Frontend plugins

Plugins can contribute UI elements to the Invect frontend:

```tsx
<Invect apiBaseUrl="/api/invect" plugins={[rbacFrontendPlugin]} />
```

Frontend plugins can add:

| Contribution    | Description                                  |
| --------------- | -------------------------------------------- |
| `sidebar`       | Navigation items in the sidebar              |
| `routes`        | New pages (e.g., `/access` for RBAC)         |
| `panelTabs`     | Tabs in the node config panel                |
| `headerActions` | Action buttons in the header (e.g., "Share") |
| `providers`     | React context providers wrapping the app     |
| `apiHeaders`    | Inject headers into all API requests         |

See [Building Plugins](/docs/plugins/building-plugins) for how to create frontend plugins.

## Styles

You must import the Invect stylesheet:

```ts
import '@invect/ui/styles';
```

All Invect CSS is scoped inside a `.invect` class — it won't conflict with your app's styles. Theme tokens use the `imp-*` prefix (e.g., `bg-imp-background`, `text-imp-foreground`).
