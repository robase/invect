## Install

Run the CLI inside an existing Express, NestJS, or Next.js project:

```bash
npx invect-cli init
```

The wizard will:

1. Detect your framework and database driver
2. Install `@invect/core`, `@invect/ui`, and the framework adapter
3. Generate `invect.config.ts` and an encryption key
4. Create database schema files and run the initial migration

<Callout type="info">
  Skip prompts with flags: `npx invect-cli init --framework express --database sqlite`
</Callout>

## Start the server

The CLI installs everything, but you still need to mount the router. Here's the minimal Express setup — the config is imported from the generated `invect.config.ts`:

```ts title="index.ts"
import express from 'express';
import { createInvectRouter } from '@invect/express';
import { invectConfig } from './invect.config';

const app = express();
app.use(express.json());
app.use('/invect', await createInvectRouter(invectConfig));

app.listen(3000, () => console.log('http://localhost:3000/invect'));
```

For NestJS or Next.js mounting patterns, see [Installation](/docs/installation).

## Open the dashboard

Start your server and visit `http://localhost:3000/invect`. You should see the Invect dashboard where you can create and run flows.

## Add the visual editor (optional)

If your frontend is a separate React app, install `@invect/ui` and mount the editor:

```tsx title="App.tsx"
import { Invect } from '@invect/ui';
import '@invect/ui/styles';

function App() {
  return <Invect apiPath="http://localhost:3000/invect" />;
}
```
