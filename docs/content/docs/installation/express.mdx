<Callout type="info">
  The fastest way to set up Invect with Express is with the CLI:
  ```bash
  npx invect-cli init --framework express
  ```
  This handles package installation, config creation, schema generation, and migration in one step. The manual steps below are for reference.
</Callout>

<Steps>

<Step>
### Install packages

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
  ```bash
  pnpm add @invect/core @invect/express
  ```
  </Tab>
  <Tab value="npm">
  ```bash
  npm install @invect/core @invect/express
  ```
  </Tab>
  <Tab value="yarn">
  ```bash
  yarn add @invect/core @invect/express
  ```
  </Tab>
  <Tab value="bun">
  ```bash
  bun add @invect/core @invect/express
  ```
  </Tab>
</Tabs>
</Step>

<Step>
### Set environment variables

```bash title=".env"
DATABASE_URL=file:./dev.db
INVECT_ENCRYPTION_KEY=   # Generate with: npx invect-cli secret
```

</Step>

<Step>
### Create the router

`createInvectRouter` is an async factory that returns an Express `Router` with all Invect API routes mounted.

```ts title="index.ts"
import 'dotenv/config';
import express from 'express';
import { createInvectRouter } from '@invect/express';

const app = express();
app.use(express.json());

const router = await createInvectRouter({
  database: {
    type: 'sqlite',
    connectionString: process.env.DATABASE_URL!,
  },
  encryptionKey: process.env.INVECT_ENCRYPTION_KEY!,
});

app.use('/invect', router);
app.listen(3000, () => {
  console.log('Invect API at http://localhost:3000/invect');
});
```

The router internally creates an `Invect` core instance, initializes the database, registers all built-in actions, and starts batch polling.

</Step>

<Step>
### Generate and migrate the database

```bash
npx invect-cli generate
npx invect-cli migrate
```

This creates Drizzle schema files for your database dialect and applies them. See [CLI Reference](/docs/reference/cli) for all options.

</Step>

<Step>
### Start the server

```bash
npx tsx index.ts
```

The API is now live at `http://localhost:3000/invect`. Add the [frontend](/docs/frontend) to get the visual flow editor.

</Step>

</Steps>

## Configuration

The `createInvectRouter` function accepts the full [`InvectConfig`](/docs/reference/configuration) object:

```ts title="index.ts"
const router = await createInvectRouter({
  database: {
    type: 'sqlite',
    connectionString: process.env.DATABASE_URL!,
  },
  encryptionKey: process.env.INVECT_ENCRYPTION_KEY!,
  execution: {
    defaultTimeout: 60000,
    maxConcurrentExecutions: 10,
  },
  triggers: {
    cronEnabled: true,
    webhookBaseUrl: 'https://your-app.com/invect',
  },
  plugins: [
    // Add plugins here
  ],
  defaultCredentials: [
    {
      name: 'OpenAI',
      type: 'llm',
      provider: 'openai',
      authType: 'api_key',
      config: { apiKey: process.env.OPENAI_API_KEY! },
    },
  ],
});
```

## With plugins

```ts title="index.ts"
import { createInvectRouter } from '@invect/express';
import { authentication } from '@invect/user-auth/backend';
import { rbacPlugin } from '@invect/rbac/backend';

const router = await createInvectRouter({
  database: {
    type: 'sqlite',
    connectionString: process.env.DATABASE_URL!,
  },
  encryptionKey: process.env.INVECT_ENCRYPTION_KEY!,
  plugins: [
    authentication({
      auth, // Your Better Auth instance
    }),
    rbacPlugin(),
  ],
});
```

After adding plugins with database tables, run `npx invect-cli generate` and `npx invect-cli migrate` again to add the plugin tables.

## PostgreSQL

Switch from SQLite by changing the database config:

```ts
const router = await createInvectRouter({
  database: {
    type: 'postgresql',
    connectionString: 'postgresql://user:pass@localhost:5432/myapp',
  },
  encryptionKey: process.env.INVECT_ENCRYPTION_KEY!,
});
```
