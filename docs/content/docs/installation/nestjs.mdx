## Packages

```bash
pnpm add @invect/core @invect/nestjs postgres
```

Use `better-sqlite3` instead of `postgres` for SQLite.

## Database setup

Create a minimal config for the CLI:

```ts title="invect.config.ts"
import { defineConfig } from '@invect/core';

export default defineConfig({
  database: {
    type: 'postgresql',
    connectionString: process.env.DATABASE_URL || 'postgresql://invect:invect@localhost:5432/invect',
  },
});
```

Generate and apply:

```bash
pnpm dlx @invect/cli generate
pnpm dlx @invect/cli migrate --push
```

## Module registration

Add `InvectModule` to your app module:

```ts title="src/app.module.ts"
import { Module } from '@nestjs/common';
import { InvectModule } from '@invect/nestjs';

@Module({
  imports: [
    InvectModule.forRoot({
      database: {
        type: 'postgresql',
        connectionString: process.env.DATABASE_URL || 'postgresql://invect:invect@localhost:5432/invect',
      },
      execution: {
        defaultTimeout: 60000,
        maxConcurrentExecutions: 10,
      },
    }),
  ],
})
export class AppModule {}
```

`InvectModule.forRoot()` registers the core as a singleton provider, mounts all API endpoints as NestJS controllers, and starts the batch polling service.

For async configuration (e.g., loading from a `ConfigService`):

```ts
InvectModule.forRootAsync({
  imports: [ConfigModule],
  inject: [ConfigService],
  useFactory: (config: ConfigService) => ({
    database: {
      type: 'postgresql',
      connectionString: config.get('DATABASE_URL'),
    },
  }),
})
```

## Accessing the core instance

Inject `InvectService` when you need to call Invect methods from your own services:

```ts title="src/my.service.ts"
import { Injectable } from '@nestjs/common';
import { InvectService } from '@invect/nestjs';

@Injectable()
export class MyService {
  constructor(private readonly invectService: InvectService) {}

  async triggerWorkflow(name: string) {
    const core = this.invectService.getCore();
    return core.startFlowRun('my-flow-id', { name });
  }
}
```

The `InvectService` is exported from the module, so any module that imports `InvectModule` can inject it.
