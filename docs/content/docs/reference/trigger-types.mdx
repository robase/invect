## Manual Trigger (`trigger.manual`)

A manual trigger marks a flow for explicit invocation — via the UI "Run" button, the REST API, or programmatic code.

### Config fields

None. The manual trigger node has no configuration.

### Behaviour

- Flow is started only when explicitly triggered.
- Input values are provided at trigger time.

## Cron Trigger (`trigger.cron`)

A cron trigger runs the flow automatically on a recurring schedule.

### Config fields

| Field            | Type     | Required | Description                                         |
| ---------------- | -------- | -------- | --------------------------------------------------- |
| `cronExpression` | `string` | Yes      | Standard 5-field cron expression.                   |
| `inputs`         | `object` | No       | Static input values passed to the flow on each run. |
| `timezone`       | `string` | No       | IANA timezone (e.g., `America/New_York`).           |

### Cron expression format

```
┌───────────── minute (0–59)
│ ┌───────────── hour (0–23)
│ │ ┌───────────── day of month (1–31)
│ │ │ ┌───────────── month (1–12)
│ │ │ │ ┌───────────── day of week (0–7, Sun=0 or 7)
│ │ │ │ │
* * * * *
```

### Examples

| Expression    | Schedule                          |
| ------------- | --------------------------------- |
| `* * * * *`   | Every minute                      |
| `*/5 * * * *` | Every 5 minutes                   |
| `0 * * * *`   | Every hour                        |
| `0 9 * * *`   | 9:00 AM daily                     |
| `0 9 * * 1-5` | 9:00 AM weekdays                  |
| `0 0 * * *`   | Midnight daily                    |
| `0 0 1 * *`   | Midnight on the 1st of each month |
| `0 0 * * 0`   | Midnight every Sunday             |

### Behaviour

- The cron scheduler starts when Invect initialises (unless `triggers.cronEnabled` is `false`).
- Each cron tick creates a new flow run with the configured static inputs.
- Missed ticks (e.g., server was down) are not retroactively executed.

## Webhook Trigger

Webhook triggers are provided by the `@invect/webhooks` plugin, not as a core trigger type. See [Add Webhooks](/docs/how-to/plugins/add-webhooks).
