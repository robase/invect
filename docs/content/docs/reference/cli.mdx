The Invect CLI (`invect-cli`) manages project initialisation, schema generation, and database migrations. It's the recommended way to set up and manage your Invect project.

```bash
npx invect-cli <command> [options]
```

<Callout type="info">
  You can also use your package manager's equivalent: `pnpm dlx invect-cli`, `yarn dlx invect-cli`, or `bunx invect-cli`.
</Callout>

All commands accept `--debug` for detailed error output and stack traces.

---

## `init`

Interactive setup wizard. Detects framework, installs dependencies, creates config, generates schemas, runs initial migration. **This is the recommended way to get started.**

```bash
npx invect-cli init
```

| Flag                     | Description                                                    |
| ------------------------ | -------------------------------------------------------------- |
| `--framework <name>`     | Skip framework detection (`express`, `nestjs`, `nextjs`).      |
| `--database <type>`      | Skip database selection (`sqlite`, `postgresql`, `mysql`).     |
| `--package-manager <pm>` | Skip package manager detection (`npm`, `pnpm`, `yarn`, `bun`). |

**Example: non-interactive setup**

```bash
npx invect-cli init --framework express --database sqlite --package-manager pnpm
```

## `generate`

Generate Drizzle schema files from core + plugin schemas.

```bash
npx invect-cli generate
```

| Flag                | Description                                                 |
| ------------------- | ----------------------------------------------------------- |
| `--config <path>`   | Path to config file (default: auto-discovery).              |
| `--dialects <list>` | Comma-separated dialects (`sqlite`, `postgresql`, `mysql`). |
| `--out-dir <dir>`   | Output directory for generated files.                       |
| `--adapter <name>`  | Schema adapter (`drizzle` or `prisma`).                     |
| `--schema-only`     | Generate schema files without migration files.              |
| `-y, --yes`         | Skip confirmation prompts.                                  |

The CLI reads `invect.config.ts`, discovers plugins, merges schemas, and generates dialect-specific Drizzle files. Unchanged files are skipped.

**Run this after adding or updating plugins with database tables.**

---

## `migrate`

Apply database schema changes.

```bash
# Production: generate + apply migration files
npx invect-cli migrate

# Development: push schema directly (fast, no migration files)
npx invect-cli migrate --push
```

| Flag              | Description                                        |
| ----------------- | -------------------------------------------------- |
| `--config <path>` | Path to config file.                               |
| `--push`          | Use `drizzle-kit push` (fast, no migration files). |
| `-y, --yes`       | Skip confirmation prompts.                         |

---

## `info`

Display diagnostic information about your setup.

```bash
npx invect-cli info
```

| Flag              | Description          |
| ----------------- | -------------------- |
| `--config <path>` | Path to config file. |
| `--json`          | Output as JSON.      |

Shows system info, detected framework, database type, config location, installed packages, and registered plugins.

---

## `secret`

Generate a cryptographically secure 32-byte base64 encryption key for `INVECT_ENCRYPTION_KEY`.

```bash
npx invect-cli secret
```

Copy the output into your `.env` file or secrets manager.

## `mcp`

Start a Model Context Protocol (MCP) server over stdio. Connects to a running Invect instance and exposes flow building/execution tools to AI assistants like Claude, Copilot, or Cursor.

```bash
npx invect-cli mcp --url http://localhost:3000/invect
```

| Flag             | Description                                                  |
| ---------------- | ------------------------------------------------------------ |
| `--url <url>`    | URL of the running Invect API (default: `http://localhost:3000/invect`). |
| `--api-key <key>` | API key for authenticated access.                           |
| `--print-config` | Output ready-to-paste MCP client config (e.g., for Claude Desktop). |

**Example: generate config for Claude Desktop**

```bash
npx invect-cli mcp --print-config
```

## Common workflows

### Initial setup

```bash
npx invect-cli init
```

### After adding a plugin with database tables

```bash
npx invect-cli generate
npx invect-cli migrate --push
```

### Production database migration

```bash
npx invect-cli generate
npx invect-cli migrate
```

### Generate encryption key

```bash
npx invect-cli secret
```
