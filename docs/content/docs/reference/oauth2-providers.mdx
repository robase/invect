title: OAuth2 Providers
description: All supported OAuth2 providers, scopes, and capabilities.
---

Invect supports 35 OAuth2 providers across 9 categories. Each provider defines authorization and token URLs, default scopes, and whether token refresh is supported.

## Google

| Provider | ID | Default Scopes | Refresh |
|----------|----|---------------|---------|
| Gmail | `google_gmail` | `gmail.readonly`, `gmail.send`, `gmail.modify` | Yes |
| Google Docs | `google_docs` | `documents`, `documents.readonly` | Yes |
| Google Sheets | `google_sheets` | `spreadsheets`, `spreadsheets.readonly` | Yes |
| Google Drive | `google_drive` | `drive`, `drive.file`, `drive.readonly` | Yes |
| Google Calendar | `google_calendar` | `calendar`, `calendar.events`, `calendar.readonly` | Yes |
| Google Analytics | `google_analytics` | `analytics.readonly` | Yes |

All Google providers use `access_type=offline` and `prompt=consent` by default to obtain refresh tokens.

## Notes

- Providers marked "API key auth" don't use OAuth2 but are included for completeness — they use API key credentials instead.
- Scopes shown are defaults. You can request additional scopes when connecting.
- Providers without refresh token support require re-authorization when the access token expires.
