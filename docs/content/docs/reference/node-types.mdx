## Core nodes

### Input (`core.input`)

Defines the flow's input parameters. When the flow runs, users provide values for each defined field.

**Config**: A list of named fields with types (`string`, `number`, `boolean`, `object`).

**Output**: An object with each field name as a key.

### Output (`core.output`)

Defines the flow's return value. Captures whatever the upstream node produces.

**Config**: None — passthrough.

**Output**: The upstream node's output.

### Model (`core.model`)

Sends a prompt to an LLM (OpenAI or Anthropic) and returns the response.

**Config**: `credential` (LLM API key), `model`, `prompt`, `systemPrompt`, `temperature`, `maxTokens`.

**Output**: `{ content, usage: { promptTokens, completionTokens, totalTokens }, model }`.

### Template String (`core.template_string`)

Evaluates a JavaScript template string against the incoming data context.

**Config**: `template` — a string with `{{ }}` expressions.

**Output**: The evaluated string (pure expression) or interpolated string (mixed).

### JavaScript (`core.javascript`)

Runs custom JavaScript code in a QuickJS WASM sandbox.

**Config**: `code` — JavaScript code. Upstream outputs available as local variables.

**Output**: Whatever the code returns.

### If/Else (`core.if_else`)

Evaluates a condition and routes to one of two branches.

**Config**: `condition` — a JavaScript expression returning boolean.

**Output handles**: `true_output`, `false_output`. The node is passthrough — output equals input.

### Switch (`core.switch`)

Evaluates multiple conditions and routes to the first match.

**Config**: A list of cases, each with a label and condition. Plus a default branch.

**Output handles**: One per case plus default. Passthrough.

## HTTP

### HTTP Request (`http.request`)

Makes an HTTP request to an external API.

**Config**: `url`, `method`, `headers`, `body`, `credential`.

**Output**: `{ data, status, headers }`.

## Triggers

### Manual Trigger (`trigger.manual`)

Marks a flow as manually triggered. No configuration.

### Cron Trigger (`trigger.cron`)

Runs the flow on a schedule.

**Config**: `cronExpression`, `inputs` (static input values).

## Agent

### Agent (`AGENT`)

Runs an iterative prompt → tool-call → response loop.

**Config**: `credential`, `model`, `taskPrompt`, `systemPrompt`, `enabledTools`, `maxIterations`, `stopCondition`, `enableParallelTools`.

**Output**: The agent's final response (text or last tool result, depending on stop condition).

## Integration nodes

All integration actions (Gmail, Slack, GitHub, etc.) follow a standard pattern:

- **Config**: Action-specific parameters plus a credential selector.
- **Output**: The API response data.

See [Built-in Actions](/docs/reference/built-in-actions) for the complete list.
