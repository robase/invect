The fastest way to get started is with the Invect CLI. It detects your framework, installs packages, generates database schemas, and runs migrations — all in one command.

## Option A: Use the CLI (recommended)

```bash
npx invect-cli init
```

The interactive wizard will:

1. Detect your framework (Express, NestJS, or Next.js)
2. Ask for your database type (SQLite, PostgreSQL, or MySQL)
3. Install `@invect/core` and the right framework adapter
4. Create `invect.config.ts`
5. Generate database schema files
6. Run the initial migration

After it finishes, start your server and open the Invect dashboard.

<Callout type="info">
  You can skip prompts with flags: `npx invect-cli init --framework express --database sqlite`
</Callout>

## Option B: Manual setup (Express + SQLite)

If you prefer to set things up step by step, this section walks through Express with SQLite. For NestJS or Next.js, see [Installation](/docs/installation).

### Install the packages

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">```bash pnpm add @invect/core @invect/express ```</Tab>
  <Tab value="npm">```bash npm install @invect/core @invect/express ```</Tab>
  <Tab value="yarn">```bash yarn add @invect/core @invect/express ```</Tab>
  <Tab value="bun">```bash bun add @invect/core @invect/express ```</Tab>
</Tabs>

### Set environment variables

Create a `.env` file in your project root:

```bash title=".env"
# Database URL
DATABASE_URL=file:./dev.db

# Encryption key for credential storage (AES-256-GCM, base64, 32 bytes)
# Generate one with: npx invect-cli secret
INVECT_ENCRYPTION_KEY=
```

Generate an encryption key:

```bash
npx invect-cli secret
```

Copy the output into your `.env` file.

### Create the server

```ts title="index.ts"
import 'dotenv/config';
import express from 'express';
import { createInvectRouter } from '@invect/express';

const app = express();
app.use(express.json());

const router = await createInvectRouter({
  database: {
    type: 'sqlite',
    connectionString: process.env.DATABASE_URL!,
  },
  encryptionKey: process.env.INVECT_ENCRYPTION_KEY!,
});

app.use('/invect', router);
app.listen(3000, () => {
  console.log('Invect running at http://localhost:3000/invect');
});
```

### Generate database tables

```bash
npx invect-cli generate
npx invect-cli migrate
```

The CLI reads your `invect.config.ts`, generates Drizzle schema files for your database dialect, and applies them. See [CLI Reference](/docs/reference/cli) for all options.

### Start the server

```bash
npx tsx index.ts
```

Open `http://localhost:3000/invect` — you should see the Invect dashboard.

<Callout type="info">
  The backend serves both the REST API and the flow data. To add the visual editor frontend, see
  [Frontend](/docs/frontend).
</Callout>

### Add the frontend (optional)

If you want the visual flow editor in a separate React app:

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">```bash pnpm add @invect/ui ```</Tab>
  <Tab value="npm">```bash npm install @invect/ui ```</Tab>
  <Tab value="yarn">```bash yarn add @invect/ui ```</Tab>
  <Tab value="bun">```bash bun add @invect/ui ```</Tab>
</Tabs>

```tsx title="App.tsx"
import { Invect } from '@invect/ui';
import '@invect/ui/styles';

function App() {
  return <Invect apiBaseUrl="http://localhost:3000/invect" />;
}
```
