Invect is a workflow orchestration library that mounts directly into your existing Node.js application. It provides a visual flow editor, AI agent nodes, 190+ built-in integrations, and batch processing — all as npm packages, not a separate platform.

## Getting started

The fastest way to set up Invect is with the CLI:

```bash
npx invect-cli init
```

This interactive wizard detects your framework, installs the right packages, creates your config, generates the database schema, and runs the initial migration — all in one command.

See the [Quickstart](/docs/getting-started/quickstart) for a full walkthrough.

## How it works

Run `npx invect-cli init` to scaffold your project, or install manually and mount a router. Open the visual editor. Build workflows that connect your AI models, APIs, and data — then trigger them from your code or on a schedule.

```ts title="index.ts"
import express from 'express';
import { createInvectRouter } from '@invect/express';

const app = express();
const router = await createInvectRouter({
  database: { type: 'sqlite', connectionString: 'file:./dev.db' },
  encryptionKey: process.env.INVECT_ENCRYPTION_KEY!,
});
app.use('/invect', router);
app.listen(3000);
```

That's it. You now have:

- A visual flow editor at `http://localhost:3000/invect`
- A full REST API for managing and executing workflows
- AI agent support with tool calling
- 190+ integration nodes ready to use

## Architecture

Invect uses a **core + adapter** pattern. All business logic lives in `@invect/core`. Framework packages (`@invect/express`, `@invect/nestjs`, `@invect/nextjs`) are thin adapters that wrap the core.
