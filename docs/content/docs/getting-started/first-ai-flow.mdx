This builds on the greeting flow from the previous page. Instead of a static template, you'll use an LLM to generate a personalised message based on the user's name and interests.

## Setting up credentials

Before using AI models, you need to store an API key. Go to **Credentials** in the sidebar, click **Add Credential**, and create one with:

- **Type**: `llm`
- **Provider**: `openai` or `anthropic`
- **Auth Type**: API Key
- **API Key**: your key

Credentials are encrypted at rest with AES-256-GCM. You can also pre-configure them in your `invect.config.ts` via `defaultCredentials` — useful for development.

## Adding the Model node

Open the Greeting Flow. Add a second input field (`topic`, type `string`) to the Input node, then drag a **Model** node from the palette onto the canvas.

Wire it as: **Input → Model → Output** (you can remove the Template String node or keep it downstream of the Model).

In the Model node's config panel, set:

- **Credential**: your API key
- **Model**: e.g., `gpt-4o-mini` or `claude-sonnet-4-20250514`
- **Prompt**: `Write a friendly, one-paragraph welcome message for {{ input.name }} who is interested in {{ input.topic }}.`

The prompt uses the same `{{ }}` template syntax. It resolves against the upstream Input node's output before being sent to the LLM.

## How Model nodes work

When the flow runs, the Model node:

1. Resolves all template expressions in the prompt
2. Sends the prompt to the configured LLM provider
3. Returns the response text as its output

Downstream nodes can reference the result via `{{ model.content }}` (where `model` is the node's reference ID). The execution inspector shows the full prompt sent, the model's response, token usage, and latency.

If the model is configured for batch mode, the node returns a `PENDING` state instead, and the flow pauses until the batch completes. See [running flows programmatically](/docs/flows/running-programmatically) for more on batch processing.
