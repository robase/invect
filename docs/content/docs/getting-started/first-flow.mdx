This guide assumes you've completed the [Quickstart](/docs/getting-started/quickstart) and have Invect running locally.

## What you'll build

A simple flow that takes a topic as input, generates a summary using an AI model, and outputs the result. The flow has three nodes:

<Mermaid chart={`
graph LR
  Input --> Model --> Output
`} />

<Steps>

<Step>
### Create a flow

Open the Invect dashboard and click **New Flow**. Give it a name like "Topic Summarizer".

You'll be taken to the visual flow editor — a blank canvas where you'll add nodes.

</Step>

<Step>
### Add an Input node

Click the **+** button in the node palette (left sidebar) and select **Input** under the Core section.

Click the node to open its config panel. Add one input field:

| Field   | Value               |
| ------- | ------------------- |
| Name    | `topic`             |
| Type    | `string`            |
| Default | `Quantum computing` |

The Input node defines the data your flow expects when triggered. Downstream nodes access these values via template expressions like `{{ input.topic }}`.

</Step>

<Step>
### Add a Model node

Add a **Model** node from the palette. Connect the Input node's output handle to the Model node's input handle by dragging between them.

Open the Model node config and set:

| Field      | Value                                                                                                     |
| ---------- | --------------------------------------------------------------------------------------------------------- |
| Credential | Select an OpenAI or Anthropic API key (create one in [Credentials](/docs/credentials/api-keys) if needed) |
| Model      | `gpt-4o-mini` or `claude-sonnet-4-20250514`                                                               |
| Prompt     | `Summarize the following topic in 3 sentences: {{ input.topic }}`                                         |

The `{{ input.topic }}` expression is resolved at runtime using the Input node's output. Template expressions support full JavaScript — see [Templates](/docs/flows/templates).

</Step>

<Step>
### Add an Output node

Add an **Output** node and connect the Model node to it.

The Output node captures the final result of your flow. It automatically receives the upstream node's data.

</Step>

<Step>
### Run the flow

Click the **Run** button in the top-right corner. The flow executes each node in topological order:

1. **Input** — produces `{ topic: "Quantum computing" }`
2. **Model** — resolves the prompt template, calls the AI model, returns the summary
3. **Output** — captures the final result

You can see the execution trace for each node by clicking on it after the run completes. Each node shows its input data, resolved parameters, and output.

</Step>

</Steps>

## Key concepts

Now that you've built a flow, here's what's happening under the hood:

### Template expressions

Any config field can contain `{{ expression }}` blocks. These are evaluated as JavaScript in a sandboxed QuickJS WASM environment. The context includes all upstream node outputs keyed by their reference ID.

```
{{ input.topic }}                          // Simple property access
{{ input.items.filter(i => i.active) }}    // Full JavaScript
{{ JSON.stringify(api_result, null, 2) }}  // Formatting
```

### Execution order

Invect uses topological sorting to determine node execution order. Nodes run in dependency order — a node only executes after all its upstream nodes have completed.

### Node data flow

Each node's output is available to downstream nodes. When building the input data for a node, Invect creates an object where:

- **Keys** are the upstream node's `referenceId` (snake_case label)
- **Values** are the upstream node's output

```json
{
  "input": { "topic": "Quantum computing" },
  "model": "Quantum computing is a type of computation..."
}
```
