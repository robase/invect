The **Agent** node runs an iterative AI loop: send a prompt to an LLM, the model calls tools, Invect executes them and returns results, and the model continues until it reaches a conclusion. This is different from the [Model](/docs/nodes/core-nodes#model) node which makes a single API call.

At each iteration, the LLM can:

- **Call one or more tools** вҖ” Invect executes them and feeds results back
- **Return a text response** вҖ” The agent loop ends
- **Call the stop tool** вҖ” Explicitly signals completion (when using `explicit_stop`)

## Quick example

Add an Agent node to your flow, configure it with:

| Field          | Value                                                           |
| -------------- | --------------------------------------------------------------- |
| Credential     | An OpenAI or Anthropic API key                                  |
| Model          | `gpt-4o` or `claude-sonnet-4-20250514`                          |
| Task Prompt    | `Research the company {{ input.company }} and create a summary` |
| Tools          | Enable `http.request`, `core.jq`                                |
| Max Iterations | `10`                                                            |
| Stop Condition | `explicit_stop`                                                 |

The agent will:

1. Plan how to research the company
2. Use HTTP Request to fetch data from APIs
3. Use JQ to parse and extract relevant info
4. Compile findings into a summary
5. Call the stop tool when done

## LLM providers

Agents support three providers:

| Provider   | Models                                                                   |
| ---------- | ------------------------------------------------------------------------ |
| OpenAI     | `gpt-4o`, `gpt-4o-mini`, `gpt-4-turbo`, `o1`, `o3-mini`                  |
| Anthropic  | `claude-sonnet-4-20250514`, `claude-sonnet-4-20250514`, `claude-3-haiku` |
| OpenRouter | Routes to multiple providers (DeepSeek, Llama, Mistral, etc.)            |

Each provider's tool calling format is handled automatically by the adapter layer.

## Stop conditions

| Condition        | Behavior                                                                                                        |
| ---------------- | --------------------------------------------------------------------------------------------------------------- |
| `explicit_stop`  | Agent must call the stop tool to end. Best for complex tasks where you want the agent to decide when it's done. |
| `tool_result`    | Agent stops after executing the first tool. Good for simple lookup-and-return patterns.                         |
| `max_iterations` | Agent runs until hitting the iteration limit. Fallback вҖ” always active regardless of selected condition.        |

## Template expressions in prompts

Agent prompts support the same [template expressions](/docs/flows/templates) as all other nodes:

```
Research {{ input.company }} and find:
1. Their annual revenue
2. Number of employees
3. Key products

Use the data from {{ api_context.industry_report }} as background.
```

## Parallel tool execution

When `enableParallelTools` is enabled, the agent can request multiple tools in a single response. Invect executes them concurrently and returns all results at once. This is more efficient for models that support it (GPT-4o, Claude 3.5+).
