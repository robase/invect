---
title: Configuration
description: Configure agent nodes — task prompt, tools, stop conditions, and model settings.
---

Agent nodes have more configuration than other node types because they control an iterative LLM loop. Here's what each setting does.

## Core settings

**Credential** — the LLM API key (OpenAI, Anthropic, or OpenRouter). Same credentials used by Model nodes.

**Model** — which model runs the loop. More capable models (GPT-4o, Claude Sonnet) produce better tool-calling decisions. Smaller models are faster and cheaper but may struggle with complex tool sequences.

**Task prompt** — what the agent should accomplish. This is the initial prompt sent to the LLM. Supports `{{ }}` template expressions, so you can incorporate data from upstream nodes:

```
Research {{ input.topic }} and compile a summary.
Use the HTTP tool to fetch relevant pages and the JQ tool to extract key data.
```

**System prompt** (optional) — sets the agent's behaviour, persona, or constraints. Good for things like "Always verify data before reporting it" or "Respond in JSON format."

## Stop conditions

The agent loop needs to know when to stop. Three options:

**`explicit_stop`** — the LLM must produce a text response (not a tool call) to end the loop. This gives the model control over when it's finished. It can keep calling tools until it's satisfied with the results.

**`tool_result`** — the loop ends after the first tool call completes. Useful when you want the agent to pick one tool and return its result immediately. Essentially makes the agent a "smart router" that chooses which tool to call.

**`max_iterations`** — the loop always runs for the full configured number of iterations, then returns the last result.

## Iteration limit

**Max iterations** — a safety cap on the loop (1–50, default 10). Even with `explicit_stop`, the loop ends if this limit is reached. Set it based on task complexity:

- Simple routing (pick a tool): 1–3
- Multi-step tasks: 5–15
- Complex research: 10–30

Each iteration is a full LLM round-trip, so higher limits mean more cost and latency.

## Parallel tools

**Enable parallel tools** — when the LLM requests multiple tool calls in a single response (e.g., "fetch these three URLs simultaneously"), this setting controls whether they execute in parallel or sequentially. Parallel execution is faster but uses more resources.

## Output

The agent node's output is the final result of the loop — either the LLM's text response (for `explicit_stop`) or the last tool result (for `tool_result` or `max_iterations`). Downstream nodes access it via the agent's reference ID like any other node.
